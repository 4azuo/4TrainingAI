📚 Ngoài làm app — tổ chức mọi project bằng agent

"Ngoài coding" không chỉ là làm app. Cùng bộ công cụ vibe coding (IDE + agent + file + sub-agent) có thể dùng để tự tổ chức bất kỳ project tri thức nào: viết tiểu thuyết, nghiên cứu, biên soạn tài liệu, dựng giáo trình…

Vì sao agent coding lại dùng được cho việc không phải code?

Vì bản chất các agent (Claude Code, Cursor…) chỉ làm 3 việc: đọc/ghi file, chạy task nhiều bước, và điều phối sub-agent theo vai. Bất kỳ "project" nào gồm nhiều file văn bản + nhiều bước + cần phối hợp đều hợp — dù nội dung là code hay chữ.

🧩 Một project = một thư mục đầy file. Code hay truyện đều vậy. Agent không phân biệt — nó đọc file, làm theo yêu cầu, ghi lại. Vậy nên "vibe writing" cũng thật như "vibe coding".

Ví dụ: viết một cuốn tiểu thuyết bằng agent

Bạn tổ chức project thành các file, rồi giao cho các sub-agent phù hợp từng vai:

tieu-thuyet/
├── the-gioi.md        # world-building: bối cảnh, quy tắc
├── nhan-vat.md        # hồ sơ nhân vật, động cơ
├── outline.md         # dàn ý tổng + theo chương
├── chuong/
│   ├── 01.md
│   └── 02.md
└── GUIDE.md           # giọng văn, ngôi kể, quy ước (như "system prompt")

Đây chính là mô hình orchestrator + sub-agent của coding, áp cho viết lách: mỗi agent giỏi một vai, bạn điều phối và duyệt kết quả.

Tự tổ chức project — chọn agent phù hợp từng vai

Loại projectCấu trúc fileCác vai (sub-agent) phù hợp

Cùng vòng lặp — áp dụng khắp nơi

Ngoài agent IDE, các app AI khác cũng dùng đúng mẫu "mô tả → AI làm → mình duyệt":

✍️ Viết & nội dung

ChatGPT, Claude, Gemini, Notion AI — soạn, biên tập, dịch, tóm tắt.

📊 Dữ liệu

AI trong Excel/Sheets, code interpreter — hỏi bằng lời, kiểm số liệu.

🎨 Thiết kế

Figma AI, Canva, Midjourney — prompt hình ảnh, tinh chỉnh.

⚙️ Tự động hóa

Zapier, Make, n8n + AI — mô tả luồng, nối tool, giám sát.

⚠️ Giới hạn không đổi: AI vẫn có thể bịa, vẫn cần ngữ cảnh đúng, và bạn vẫn là người chịu trách nhiệm cuối — dù là code, một chương truyện hay một báo cáo.
