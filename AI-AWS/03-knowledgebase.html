📚 Bedrock Knowledge Bases — RAG được quản lý

RAG = trả lời dựa trên tài liệu của bạn. Knowledge Bases lo trọn khâu: chia nhỏ tài liệu → tạo embedding → lưu vector → truy hồi → ghép vào prompt. Bạn chỉ trỏ tới dữ liệu trên S3.

Knowledge Bases làm gì cho bạn

  • Ingestion: đọc nguồn (S3, web crawler, Confluence, SharePoint, Salesforce), chunk tài liệu.
  • Embedding: gọi model embedding (Titan Embeddings / Cohere) sinh vector.
  • Lưu trữ vector: ghi vào OpenSearch Serverless, Aurora pgvector, Pinecone, Redis, Neptune...
  • Retrieve & Generate: 1 API RetrieveAndGenerate truy hồi đoạn liên quan + gọi LLM trả lời kèm trích dẫn nguồn.
