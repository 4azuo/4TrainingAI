🤖 Amazon Bedrock Agents — LLM biết hành động

Agent = LLM + khả năng làm việc. Không chỉ trả lời, Agent lập kế hoạch nhiều bước, gọi API/Lambda của bạn, tra Knowledge Base, rồi tổng hợp kết quả — tự động dàn xếp (orchestration) thay bạn.

Khi nào dùng Agent thay vì gọi model thường?

✅ Dùng Agent khi

  • Tác vụ nhiều bước, cần gọi nhiều API theo thứ tự.
  • Cần model tự quyết nên gọi công cụ nào.
  • Muốn kết hợp RAG + hành động trong một luồng.

⚠️ Không cần Agent khi

  • Chỉ hỏi-đáp đơn giản → gọi thẳng FM.
  • Chỉ cần tra tài liệu → Knowledge Base + RetrieveAndGenerate.
  • Luồng cố định, không cần model tự quyết → tự code/Step Functions.
🧩 Ví dụ tích hợp — Trợ lý đặt lịch & kiểm tra đơn
Bedrock Agent với 2 Action Group: "tra đơn hàng" (Lambda → DynamoDB) và "đặt lịch" (Lambda → API lịch); gắn Knowledge Base chính sách đổi-trả trên S3; Guardrail lọc PII; toàn bộ gọi qua API Gateway.
📌 So với LangChain tự dựng: Bedrock Agents là managed orchestration — bạn không tự viết vòng lặp ReAct, AWS lo phần gọi tool, parse, retry.
