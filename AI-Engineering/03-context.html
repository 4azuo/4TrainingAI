🧠 Cấp 2 — Context Engineering

Prompt giỏi mà AI thiếu thông tin thì vẫn sai. Context Engineering là nghệ thuật đưa đúng thông tin, vừa đủ, vào "tầm nhìn" của AI (context window) tại đúng thời điểm.

Context window — "bàn làm việc" của AI

AI chỉ "thấy" những gì nằm trong context window của lượt đó. Cửa sổ này có hạn: nhồi quá nhiều → tốn token, chậm, và AI dễ "lạc" giữa biển thông tin (mất tập trung vào phần quan trọng). Mục tiêu không phải nhiều ngữ cảnh, mà là đúng ngữ cảnh.

Các nguồn ngữ cảnh bạn có thể "nạp"

📄 File & code trực tiếp

Đưa thẳng đoạn code/tài liệu liên quan. Chính xác nhất, nhưng tốn chỗ — chỉ đưa phần cần.

📚 RAG (truy hồi)

Tự động tìm & chèn đoạn tài liệu liên quan từ kho riêng. Hợp khi dữ liệu lớn, không thể nhồi hết.

💾 Memory (bộ nhớ)

Ghi nhớ sự thật/sở thích qua nhiều phiên (vd file CLAUDE.md, memory của agent). Bền vững, gọn.

🔌 Tool / MCP

Để AI tự lấy ngữ cảnh khi cần (đọc file, query DB, gọi API) thay vì nạp sẵn tất cả.

Đưa ngữ cảnh đúng cách

✅ Nên

  • Chọn lọc: chỉ phần liên quan trực tiếp.
  • Cấu trúc rõ: tiêu đề, đường dẫn file, ranh giới đoạn.
  • Đặt quy ước dự án vào "bộ nhớ" (CLAUDE.md / system prompt) để khỏi lặp lại.
  • Để agent tự đọc file qua tool khi cần, thay vì dán cả repo.

❌ Tránh

  • Dán nguyên cả codebase "cho chắc".
  • Để rác/ngữ cảnh cũ tích tụ làm AI lẫn lộn.
  • Trộn nhiều task không liên quan trong 1 phiên.
  • Quên rằng ngữ cảnh quá dài làm AI "quên" phần đầu.

💡 Câu thần chú: "Right context, right time, minimal noise." Đưa đúng thứ cần, đúng lúc, càng ít nhiễu càng tốt.

Khi ngữ cảnh đã chuẩn nhưng việc cần nhiều bước, tự sửa lỗi — ta lên Loop Engineering (cấp 3).
