🤖 Agent-Based Development

Bước phát triển mà AI agent là thành viên thật trong đội: nhiều agent chuyên vai — lập kế hoạch, code, review, test — phối hợp dưới một orchestrator để hoàn thành việc lớn, thay vì một mình bạn ra lệnh từng bước.

Một agent vs. nhiều agent

Một agent tự lặp (Loop) đã làm được task vừa. Việc lớn hơn thường chia vai sẽ tốt hơn: mỗi agent có system prompt, tool và ngữ cảnh riêng cho đúng phần việc của nó — giống một đội người thật.

Các mẫu phối hợp hay gặp

Orchestrator → Sub-agent

Một agent điều phối chia task cho các agent con chuyên vai, rồi tổng hợp lại. Mẫu phổ biến nhất.

Song song (parallel)

Nhiều agent làm các phần độc lập cùng lúc (mỗi agent một nhánh/worktree) → nhanh hơn, ít đụng nhau.

Đường chuyền (pipeline)

Kết quả agent này là đầu vào agent kế: Plan → Code → Test → Review nối tiếp.

Tranh luận / phản biện

Agent Reviewer soi lỗi agent Coder; hai góc nhìn khác nhau giúp bắt sai sót sớm.

Phải "engineer" những gì?

Yếu tốVì sao quan trọng
Ranh giới vai rõMỗi agent một việc + tool riêng. Vai chồng chéo → dẫm chân, lãng phí token.
Giao tiếp giữa agentTruyền đúng & đủ thông tin (spec, kết quả), tránh "tam sao thất bản".
Ngân sách & giới hạnNhiều agent = nhiều token/chi phí. Đặt trần bước, trần tiền, timeout.
Gộp kết quảOrchestrator phải hợp nhất output, xử lý xung đột (merge, mâu thuẫn).
Người chốt cuốiHành động không thể hoàn tác (merge, deploy, gửi) cần con người duyệt.

✅ Khi nào nên nhiều agent

  • Việc lớn, chia được thành phần độc lập.
  • Cần nhiều góc nhìn (code vs. review).
  • Muốn chạy song song cho nhanh.
  • Mỗi phần cần tool/ngữ cảnh rất khác nhau.

❌ Khi nào đừng vội

  • Task nhỏ — một agent là đủ, thêm agent chỉ tốn kém.
  • Chưa chạy ổn bằng một agent thì đừng nhân lên.
  • Việc dính chặt vào nhau, khó chia → khó song song.
  • Chưa có cách gộp kết quả & giám sát.
⚠️ Nhiều agent nhân cả sức mạnh lẫn lỗi: một hiểu sai spec có thể lan ra cả đội, tốn token âm thầm. Bắt buộc có log từng agent, trần chi phí, và điểm gộp/duyệt của con người.
✅ Lộ trình: làm spec rõ → chạy một agent cho ổn → mới tách vai & song song hóa khi thật sự cần. Đừng bắt đầu bằng "đội 5 agent" cho một việc một người làm được.

Agent-Based Development là cách tổ chức lực lượng AI; khi gắn nó vào sự kiện thật để tự khởi động (cron, webhook, PR mới) thì chính là Workflow khép kín ở cấp 4.
